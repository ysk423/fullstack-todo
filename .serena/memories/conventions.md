# Conventions

## TDD（テスト駆動開発）— 必須ルール

CLAUDE.md で定義されたプロジェクト必須規約。すべての実装に適用する。

1. 期待される入出力に基づき、まずテストを作成する
2. 実装コードは書かず、テストのみ用意する
3. テストを実行し、**失敗を確認する**（Red フェーズ）
4. テストが正しいことを確認できた段階でコミットする
5. テストをパスさせる実装を進める（Green フェーズ）
6. 実装中はテストを変更しない。コードのみ修正する
7. すべてのテストが通過するまで繰り返す

## コードスタイル

- 言語固有の規約は tech stack 決定後に追記
- コメントは WHY が非自明な場合のみ（WHAT を説明するコメントは書かない）
- 不要な抽象化・将来のための設計はしない

## その他

- セキュリティ: SQL インジェクション・XSS 等 OWASP Top 10 を避ける
- ユーザー入力・外部 API の境界でのみバリデーション
